# KCU Coach - UI/UX Specification

## Design System

### Brand Colors
| Name | Hex | Usage |
|------|-----|-------|
| KCU Gold | #F59E0B | Primary accent, CTAs, highlights |
| KCU Gold Dark | #D97706 | Hover states, secondary accent |
| KCU Gold Muted | #92400E | Borders, subtle accents |
| Background Primary | #0D0D0D | Main background |
| Background Card | #161616 | Card backgrounds |
| Background Elevated | #1F1F1F | Elevated surfaces |
| Border Primary | #2A2A2A | Default borders |
| Success | #22C55E | Profit, bullish, positive |
| Error | #EF4444 | Loss, bearish, negative |
| Warning | #F59E0B | Alerts, forming setups |
| Info | #3B82F6 | Information, links |

### Typography
- **Display**: Anton (headers, stats)
- **Body**: Inter (all body text)
- **Mono**: JetBrains Mono (prices, code)

### Corner Radius
- **None**: Sharp corners on all elements
- This creates a professional trading platform aesthetic vs rounded "mobile app" look

### Patterns
- Hexagonal background pattern at 3% opacity
- Used on main dashboard background

## User Flows

### Flow 1: Student Views Setup and Takes Trade

```
1. Student opens Companion Mode
2. Sees admin's shared watchlist + personal symbols
3. Notices NVDA has ‚ö° indicator (high confluence)
4. Clicks NVDA to expand setup details
5. Sees L/T/P breakdown: L:85, T:90, P:80 = 85% confluence
6. Reads AI coaching note explaining the setup
7. Reviews Entry: $142.50, Stop: $141.80, Targets
8. Clicks "Alert on Break" to get notification
9. When alert fires, takes trade
10. Logs trade in Journal with screenshot
11. AI auto-grades the trade for LTP compliance
```

### Flow 2: Admin Creates Shared Watchlist

```
1. Admin opens Admin > Shared Watchlist
2. Clicks "Add Symbol" ‚Üí enters SPY, QQQ, NVDA, TSLA, AAPL
3. Toggles "Share with all students" = ON
4. Optionally adds notes per symbol
5. Clicks Save
6. All students now see these symbols in Companion
7. System begins calculating levels and detecting setups
```

### Flow 3: Super Admin Configures Strategy

```
1. Super Admin opens Super Admin > Strategy
2. Adjusts LTP Detection Thresholds
   - Increases Confluence Threshold from 70 to 75
   - Enables "Require MTF Alignment" at 60%
3. Under Key Levels, enables "Fibonacci Levels"
4. Under AI Coach, changes model to Claude Opus
5. Clicks "Save Configuration"
6. System immediately applies new thresholds
7. Some previously "Ready" setups may drop to "Forming"
```

### Flow 4: Earnings Alert Flow

```
1. System fetches earnings calendar from Massive.com API
2. Stores upcoming earnings in economic_events table
3. User subscribes to NVDA earnings (1 day before)
4. Day before: System sends notification
   "‚ö†Ô∏è NVDA reports earnings tomorrow after market close"
5. User sees warning in Companion Mode for NVDA
6. AI Coach includes earnings context in responses
7. After earnings: System fetches actual vs expected
8. Updates event with surprise data
```
